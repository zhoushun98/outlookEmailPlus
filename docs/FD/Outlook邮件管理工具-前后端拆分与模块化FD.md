## 2. 总交付清单（必须完成）

### 2.1 后端拆分交付（可维护性与边界清晰）

- 提供新的后端模块化目录结构（按职责分层），并保证各模块职责清晰：
  - 配置（环境变量/默认值）
  - 安全（加密/鉴权/CSRF）
  - 错误结构（统一错误 payload、trace_id、脱敏、全局异常处理）
  - 数据库（连接管理、初始化与迁移、索引幂等创建）
  - 审计（写入与查询）
  - 外部集成服务（Graph/IMAP/GPTMail 等）
  - 定时任务（调度器、锁、运行记录）
- 保持现有 API 行为不变（URL/方法/请求参数语义/响应结构/错误结构/trace_id 透传/默认脱敏）。
- 路由分模块组织（例如按“系统/调度器/分组/账号/标签/邮件/OAuth/设置/临时邮箱”拆分），并提供统一的路由注册入口。
- 数据访问逻辑与业务编排逻辑拆分（SQL 与业务规则避免混杂），便于后续单测与 Mock。

### 2.2 后端兼容性与部署交付（不破坏现有运行方式）

- 保持现有部署入口可用：
  - Gunicorn/Docker 仍可使用既有命令启动（例如 `web_outlook_app:app` 入口保持可用）。
- 保持本地运行方式可用（开发者按 README 的方式可启动并访问页面）。
- 允许短期存在兼容层（旧入口转发到新模块），但必须提供“可淘汰”的清晰路径（见 PRD 的阶段建议）。

### 2.3 前端拆分交付（模板瘦身 + 资源模块化）

- `templates/index.html` 仅保留页面结构与必要容器，不再承载大段内联脚本。
- 提供前端静态资源目录并模块化：
  - JS：按“基础设施（API/错误处理/trace 展示/通用 UI）+ 业务 feature（groups/accounts/tags/emails/refresh/export/settings/audit/temp_emails）”拆分。
  - CSS：至少拆出主样式文件，避免样式与脚本混在模板内。
- 保持现有 UI 与交互行为一致（不新增前端框架，不引入构建链）。
- 保证静态资源可被浏览器缓存（相较内联脚本），并具备稳定引用方式（例如版本参数或文件名策略）。

### 2.4 测试与回归交付（重构可控）

- 保证现有测试用例在拆分后仍可运行通过（不降低覆盖面）。
- 增加“拆分相关”的基础测试覆盖（至少包括）：
  - 应用装配/路由注册的烟测（关键接口可用）
  - 错误结构与 trace_id 的一致性不回归
  - 审计写入与查询入口可用
- 保持 CI 可重复执行（自动跑单测）。

### 2.5 文档交付（面向维护者）

- 更新/新增开发者文档：
  - 新目录结构说明（模块职责与依赖边界）
  - 如何新增/修改一个 API 的推荐路径（route/service/repository）
  - 前端模块拆分说明（API 层/feature 层/UI 组件层）
- 更新验收与回归文档中涉及的入口/路径（若入口变更，必须提供兼容与迁移说明）。

---

## 3. 非功能交付项（必须满足）

### 3.1 行为兼容与稳定性

- 拆分过程中任何阶段都必须保持：
  - 核心功能可用（不因“文件拆分”导致功能不可运行）
  - 接口契约稳定（不改变现有返回结构与错误结构）
  - 调度器/后台任务策略不被意外改变

### 3.2 可回滚与渐进式交付

- 每个阶段的拆分都必须可回滚（例如保留旧入口或提供兼容切换方式）。
- 每个阶段都必须有对应的验证方法（单测 + 抽样接口验收）。

### 3.3 安全与合规

- 静态资源与模板中不得引入敏感信息（密钥/令牌/密码）。
- 保持既有默认脱敏、二次验证、审计可追溯策略不回归。

### 3.4 性能与体验

- 拆分不引入明显性能退化（启动时间/关键接口耗时不显著增加）。
- 静态资源拆分后，页面加载体验不下降（必要时提供缓存与加载顺序保障）。

## 4. 验收检查表（建议用于实施每个阶段后）

- 部署入口不变：原有 Docker/Gunicorn 启动命令可用，页面可访问。
- 基础探活：`GET /healthz` 正常。
- 关键抽样接口正常：
  - `GET /api/system/health`
  - `GET /api/scheduler/status`
  - `GET /api/system/upgrade-status`
  - `GET /api/accounts`、`/api/groups`、`/api/tags`
- 单测通过：`python -m unittest discover -s tests -v`。
- 前端页面关键路径抽样回归（至少覆盖：登录、分组/账号/标签、读取/删除/导出、审计、临时邮箱、设置）。
