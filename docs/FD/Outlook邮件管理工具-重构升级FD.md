## 1. 文档目的

本 FD 用于把 PRD 中的需求拆解为“需要完成的功能清单”，方便开发、测试、验收对齐。  
仅描述**要实现/保证的功能与行为**，不包含实现方式与技术细节。

---

## 2. 总交付清单（必须完成）

### 2.1 登录与会话

- 提供登录入口与登录页面/弹窗。
- 支持密码登录进入管理界面。
- 支持退出登录。
- 支持在线修改登录密码（不少于 8 位）。
- 登录失败具备限制机制，并提示剩余等待时间。
- 重启/升级后登录体验可预期（允许重新登录，但不得出现状态错乱或异常阻塞）。

### 2.2 分组管理

- 分组列表展示（含分组名称、颜色、描述、账号数量）。
- 新增分组（含名称、颜色、描述、网络访问策略配置）。
- 编辑分组（同上字段可修改）。
- 删除分组（需避免误操作；删除后账号需有明确归属规则）。
- 系统保留分组展示与标识；系统分组需限制危险操作（例如不允许被删除或被批量迁移进入）。

### 2.3 账号管理

- 账号列表展示（含邮箱、分组、备注、状态、标签、最近刷新时间/状态）。
- 新增账号（支持录入必要字段）。
- 编辑账号（支持修改必要字段、分组、备注、状态）。
- 删除账号（按单个/批量，需明确提示不可逆风险）。
- 批量导入账号：
  - 支持粘贴/上传导入内容
  - 支持预校验与错误定位（明确指出哪一行/哪一项字段错误）
  - 支持导入结果汇总（成功/失败数量与原因）
- 批量移动分组（禁止移动到系统保留分组）。
- 账号状态管理（停用账号不参与自动刷新等后台动作，并在界面明显标识）。

### 2.4 标签管理

- 标签列表展示（含名称、颜色、使用数量可选）。
- 新增标签（名称+颜色，名称不可重复）。
- 删除标签（需确认，删除后账号标签关系同步更新）。
- 账号批量打标签/去标签（可对选中账号执行）。
- 账号按标签筛选与组合筛选（与关键词/分组筛选兼容）。

### 2.5 邮件读取与展示

- 支持至少三类文件夹查看：收件箱、垃圾邮件、已删除邮件。
- 邮件列表分页/滚动加载，具备“加载中/无更多/加载失败”状态提示。
- 邮件详情展示（主题、发件人、收件人/抄送、时间、正文）。
- 支持长邮件的完整展示与“全屏查看”。
- 支持“信任邮件模式/原始内容查看”选项：
  - 默认模式必须安全展示
  - 信任模式必须有风险提示与显著开关状态
- 邮件内容的安全展示要求：
  - 默认不允许执行脚本类内容
  - 默认不允许危险元素导致越权或跳转劫持

### 2.6 Token 刷新（手动/全量/定时）

- 单账号手动刷新：
  - 一键触发
  - 返回成功/失败与失败原因
  - 刷新结果被记录（见 2.7）
- 全量刷新：
  - 支持一键触发
  - 提供实时进度反馈（总数、当前、成功数、失败数）
  - 支持对失败账号进行重试
  - 支持仅刷新失败账号
- 定时刷新配置与运行：
  - 支持两种调度配置方式（按天数间隔、按表达式配置）
  - 支持启用/停用开关
  - 支持配置校验与未来触发时间预览
  - 必须避免“看起来已开启但实际未运行”的情况
  - 必须提供可验证的运行证据（最近一次触发时间、触发来源标识、触发结果统计）

### 2.7 刷新日志与统计

- 记录每次刷新结果（手动/全量/定时/重试）：
  - 至少包含：时间、账号、触发类型、结果、失败原因（如有）
- 提供近半年刷新历史查询与展示。
- 自动清理过期记录（默认策略可用且可解释）。
- 提供失败账号统计与快速入口（例如“查看失败列表”）。

### 2.8 邮件删除

- 支持单封删除。
- 支持批量删除。
- 删除前必须二次确认。
- 删除失败必须给出明确失败原因分类与可操作提示，并支持重试。

### 2.9 读取/刷新/删除的回退与错误展示（用户可感知）

- 对关键动作（读取邮件、刷新 Token、删除邮件）执行既定优先级与回退策略，以提升成功率。
- 当所有方式失败时：
  - 必须逐项展示每种方式的失败原因（中文可理解）
  - 必须提供统一的错误编号/追踪标识
- 当判定为明显网络/代理类故障时：
  - 必须减少无意义重试
  - 必须优先提示用户检查网络/代理配置

### 2.10 导出与二次验证

- 支持按分组导出。
- 支持全量导出。
- 支持选择导出。
- 导出前必须二次验证（再次输入密码或等效确认方式）。
- 导出文件命名清晰可识别（包含范围、时间等关键信息）。
- 导出行为必须被审计（见 2.11）。

### 2.11 审计与可追溯

- 对以下敏感操作记录审计：
  - 导出
  - 删除
  - 修改账号
  - 修改分组
  - 修改设置
- 审计记录至少包含：操作类型、操作对象、时间、来源标识（如 IP）。
- 审计失败不得影响主流程，但需可被管理员发现（例如在日志/界面提示）。

### 2.12 OAuth2 辅助

- 生成授权链接（可复制）。
- 换取凭据并展示结果（成功返回必要信息；失败展示原因）。
- 常见失败原因提示（例如回调地址不匹配、权限不足、授权码失效等）。
- 支持配置相关参数，并提供校验与提示。

### 2.13 临时邮箱

- 生成临时邮箱。
- 临时邮箱列表查看与删除。
- 临时邮箱邮件列表查看、邮件详情查看。
- 删除临时邮件、清空临时邮箱邮件。
- 失败提示需区分：服务不可用/鉴权失败/网络异常，并给出可操作建议。

### 2.14 设置中心

- 修改登录密码。
- 修改临时邮箱服务凭据。
- 修改刷新策略相关设置（间隔、调度方式、开关等）。
- 设置变更需要明确“是否即时生效”，若非即时必须给出提示。
- 对高风险设置变更提供必要风险提示（例如可能降低安全性、可能提升失败率）。

## 3. 非功能交付项（必须满足）

### 3.1 一致性与长期运行

- 重启/升级后，数据可继续使用且行为可预期。
- 并发访问下，关键操作结果一致，避免偶发性错误与状态丢失。

### 3.2 性能基线

- 账号规模达到 1,000 时，账号列表加载、筛选、关键操作仍保持可用体验。
- 全量刷新在 1,000 账号规模下，需持续输出进度与结果，不出现长时间无反馈。

### 3.3 安全基线

- 敏感信息在界面与接口中默认脱敏；仅在必要且完成二次验证的场景下允许导出/展示。
- 错误信息不得泄露敏感内容，同时必须提供追踪标识便于定位。
- 邮件 HTML 展示默认安全，禁止脚本执行与危险行为。

## 4. 验收检查表（建议用于发布前）

- 登录：成功/失败限制/修改密码/退出均正常。
- 分组：增删改查正常；系统分组保护有效；账号计数准确。
- 账号：增删改查、批量导入、批量分组移动、状态启停均正常。
- 标签：增删、批量打/去标签、筛选正常。
- 邮件：三类文件夹可读；分页/滚动；详情/全屏；默认安全展示；信任模式提示清晰。
- 刷新：单个/全量/失败重试/仅刷新失败/定时配置与运行证据均正常；日志统计可查。
- 删除：单封/批量；二次确认；失败原因清晰可操作。
- 导出：三种导出；二次验证；导出审计可追溯；文件命名符合规则。
- OAuth2：授权链接与换取流程可用；失败提示明确。
- 临时邮箱：生成/列表/邮件查看/删除/清空可用；失败提示分类正确。
- 设置：各项设置可修改且生效时机明确；高风险项有提示。
