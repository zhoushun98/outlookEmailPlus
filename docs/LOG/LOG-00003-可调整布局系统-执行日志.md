### UI 问题修复：窄屏下“全部折叠后无法再展开”（F2/F5 交互回归）

#### 现象

- 当窗口较窄（触发 `@media (max-width: 768px)`）且 LayoutManager 自动折叠/用户折叠侧边栏后：
  - **分组面板 / 账号面板**的折叠指示器会被一起移出屏幕（left 为负值），导致用户看起来“面板消失且再也打不开”。
  - 若同时触发业务侧的“隐藏邮件列表”（`.email-list-panel.hidden`），则左侧三个区域都不可恢复，交互卡死。

#### 根因定位

- `static/css/main.css` 在窄屏下仍保留历史规则：`.group-panel,.account-panel { position: fixed; left:-280px; }`
- 新的可调整布局系统（`static/js/layout-manager.js`）在折叠时把 Grid 列宽置 0，并依赖 **折叠指示器** 提供“展开入口”。
- 当历史规则把面板整体移出屏幕时，折叠指示器作为面板子元素也一并移出，造成不可展开。

#### 修复方案

- 在布局系统初始化时，为 `<html>` 添加标记类：`layout-system-enabled`。
- 将窄屏历史侧栏滑入样式限定在 “布局系统未启用” 的场景：
  - 仅对 `html:not(.layout-system-enabled)` 生效，避免影响 LayoutManager 的折叠/指示器逻辑。

#### 变更点

- `static/js/layout-bootstrap.js`：LayoutManager 初始化前添加 `layout-system-enabled`。
- `static/css/main.css`：`@media (max-width: 768px)` 内的 `.group-panel/.account-panel` 固定定位规则增加 `html:not(.layout-system-enabled)` 限定。

#### 验证

- `npm test` 通过（11 suites / 93 tests）。
- Playwright 窄屏复现验证（viewport=600）：
  - 修复前：groups/accounts 指示器 `x < 0`（不可见）
  - 修复后：groups/accounts/emails 指示器分别落在 `x=0/48/96`（可见且可点击展开）

### UI 问题修复：折叠后页面出现滚动条、指示器高度观感异常（“太长”）

#### 现象

- 在桌面宽屏下手动折叠侧边栏（如：账号/邮件列表）后，页面右侧出现 **body 滚动条**。
- 因 body 被撑出滚动高度，用户会看到主容器整体被滚动，折叠指示器与布局区域的高度/对齐出现“太长/不正确”的观感。

#### 根因定位

- 折叠时 Grid 列宽被置为 `0px`（layout-width），但我们仅用 `opacity:0` 隐藏面板内容：
  - 面板内容仍参与布局计算；
  - 在 `0` 宽度下文本/控件会被强制换行，导致折叠面板内部出现异常的“高度溢出”；
  - 溢出内容默认 `overflow: visible`，会把 body 的滚动高度撑大。

#### 修复方案

- `static/css/layout.css`：
  - `.main-container` 增加 `overflow: hidden;`，阻止溢出影响 body 滚动条；
  - `.main-container > *` 增加 `min-height: 0;`，允许 Grid 子项在固定高度下正确收缩；
  - `.resizable-panel.collapsed > :not(.collapsed-indicator)` 增加 `display: none;`，让折叠时隐藏内容**不参与布局与滚动高度计算**。

#### 验证

- Playwright 复现验证（宽屏，折叠 accounts/emails）：
  - 修复前：`documentElement.scrollHeight > clientHeight`（出现滚动条）
  - 修复后：`documentElement.scrollHeight === clientHeight`（滚动条消失），折叠指示器高度与主容器一致
