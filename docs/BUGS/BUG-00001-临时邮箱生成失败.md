## 基本信息

- **BUG编号**：BUG-00001
- **发现日期**：2026-02-23
- **报告人**：用户
- **严重程度**：P1（高优先级）
- **状态**：✅ 已修复

- **修复日期**：2026-02-23

## 问题描述

### 用户错误信息
```
生成临时邮箱失败，请稍后重试
```

### 错误详情
- **Code**: LEGACY_ERROR
- **Type**: LegacyError
- **Status**: 400
- **Trace ID**: 557910def72a4ea7a59c8b6fb7880a98

## 复现步骤

1. 登录系统
2. 进入临时邮箱管理页面
3. 点击"生成临时邮箱"按钮
4. 系统返回错误提示

## 预期行为

- 成功生成临时邮箱地址
- 返回邮箱地址给用户
- 在数据库中记录该临时邮箱

## 实际行为

- 生成失败
- 返回笼统的错误信息
- 用户无法知道具体失败原因

## 技术分析

### 相关代码位置

1. **路由层**：`outlook_web/routes/temp_emails.py:9`
   - 路由：`/api/temp-emails/generate`
   - 方法：POST

2. **业务逻辑层**：`outlook_web/legacy.py:4171-4186`
   ```python
   def api_generate_temp_email():
       """生成新的临时邮箱"""
       data = request.json or {}
       prefix = data.get('prefix')
       domain = data.get('domain')

       email_addr = generate_temp_email(prefix, domain)

       if email_addr:
           if add_temp_email(email_addr):
               log_audit('create', 'temp_email', email_addr, "生成临时邮箱")
               return jsonify({'success': True, 'email': email_addr, 'message': '临时邮箱创建成功'})
           else:
               return jsonify({'success': False, 'error': '邮箱已存在'})
       else:
           retursonify({'success': False, 'error': '生成临时邮箱失败，请稍后重试'})
   ```

3. **服务层**：`outlook_web/services/gptmail.py:42-57`
   ```python
   def generate_temp_email(prefix: str = None, domain: str = None) -> Optional[str]:
       """生成临时邮箱地址"""
       json_data = {}
       if prefix:
           json_data["prefix"] = prefix
       if domain:
           json_data["domain"] = domain

       if json_data:
           result = gptmail_request("POST", "/api/generate-email", json_data=json_data)
       else:
           result = gptmail_request("GET", "/api/generate-email")

       if result and result.get("success"):
           return result.get("data", {}).get("email")
       return None
   ```

4. **错误处理**：`outlook_web/legacy.py:253-267`
   - 将字符串错误转换为 LEGACY_ERROR

### 根本原因分析

1. **直接原因**：`generate_temp_email()` 函数返回 `None`

2. **可能的根本原因**：
   - ❌ GPTMail API Key 未配置或无效
   - ❌ GPTMail API 服务不可用
   - ❌ 网络连接问题
   - ❌ API 请求超时（30秒）
   - ❌ API 返回数据格式不符合预期

3. **错误处理问题**：
   - ❌ `gptmail_request` 返回的详细错误信息被丢弃
   - ❌ `generate_temp_email` 只返回 None，不返回错误原因
   - ❌ `api_generate_temp_email` 使用笼统的错误信息
   - ❌ 没有记录详细的错误日志

### 错误传播链

```
GPTMail API 失败
    ↓
gptmail_request() 返回 {"success": False, "error": "具体错误"}
    ↓
generate_temp_email() 返回 None（错误信息丢失）
    ↓
api_generate_temp_email() 返回 "生成临时邮箱失败，请稍后重试"
    ↓
错误处理中间件转换为 LEGACY_ERROR
    ↓
用户看到笼统的错误信息
```

2. **修改 `api_generate_temp_email` 函数**：

## 修复详情

### 1. 改进 `gptmail_request` 函数（outlook_web/services/gptmail.py）

**改进内容**：
- 添加 API Key 配置检查
- 区分 10+ 种错误类型：
  - `CONFIG_ERROR`：API Key 未配置
  - `AUTH_ERROR`：API Key 无效（401）
  - `PERMISSION_ERROR`：访问被拒绝（403）
  - `RATE_LIMIT_ERROR`：请求频率超限（429）
  - `SERVER_ERROR`：服务器错误（500+）
  - `HTTP_ERROR`：其他 HTTP 错误
  - `TIMEOUT_ERROR`：请求超时
  - `CONNECTION_ERROR`：网络连接失败
  - `REQUEST_ERROR`：其他请求异常
  - `METHOD_ERROR`：不支持的请求方法
  - `UNKNOWN_ERROR`：未知错误
- 返回结构化错误信息：`{success, error, error_type, details}`

### 2. 改进 `generate_temp_email` 函数（outlook_web/services/gptmail.py）

**改进内容**：
- 返回值从 `Optional[str]` 改为 `Tuple[Optional[str], Optional[str]]`
- 成功时返回 `(email_addr, None)`
- 失败时返回 `(None, error_message)`，保留详细错误信息
- 添加数据格式验证

### 3. 改进 `api_generate_temp_email` 函数（outlook_web/legacy.py）

**改进内容**：
- 解包 `generate_temp_email` 的返回值：`email_addr, error_msg = generate_temp_email(...)`
- 使用详细的错误信息返回给前端
- 添加日志记录（成功、警告、错误三个级别）

### 4. 更新前端错误提示（static/js/features/temp_emails.js）

**改进内容**：
- 检查错误对象的结构
- 优先显示 `error.message` 或 `error.error`
- 支持结构化错误对象和字符串错误
- 保留错误详情用于调试

### 5. 编写单元测试（tests/test_gptmail_error_handling.py）

**新增测试**：
- 11 个新的单元测试，覆盖所有错误场景
- 使用 mock 模拟各种失败情况
- 验证错误类型和错误信息的正确性
