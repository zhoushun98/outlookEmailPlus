# BUG-00002 侧边栏折叠与页面加载问题

**日期：** 2026-03-01  
**来源：** 用户截图反馈 (PixPin_2026-03-01_19-14-29.png)  
**关联：** PRD-00004 UI全局美化重设计  

---

## 问题清单

### BUG-012：侧边栏折叠后导航图标消失 🔴 HIGH

**现象：** 点击 ☰ 按钮折叠侧边栏后，导航菜单项完全不可见，只剩一个 logo 图标。用户无法通过图标切换页面。

**根因分析：**
CSS 选择器 `.sidebar-collapsed .sidebar .nav-item span` 匹配了**所有** `<span>`，包括承载 emoji 图标的 `<span class="nav-icon">`。

```css
/* 当前（错误） */
.sidebar-collapsed .sidebar .nav-item span { display: none; }

/* 应改为 */
.sidebar-collapsed .sidebar .nav-item span:not(.nav-icon) { display: none; }
```

**HTML 结构参考：**
```html
<button class="nav-item">
    <span class="nav-icon">🏠</span>  ← 应保留
    <span>仪表盘</span>              ← 应隐藏
</button>
```

**修复方案：**
- 修改 `static/css/main.css` 中 `.sidebar-collapsed` 相关选择器，排除 `.nav-icon`
- 折叠时 `.nav-item` 应 `justify-content: center` 让图标居中

### BUG-013：账号管理页面仍显示"临时邮箱"分组 🟡 MEDIUM

**现象：** 临时邮箱已有独立页面，但在"账号管理"的分组列表中仍然显示"临时邮箱"分组，点击后仍在 accountList 中渲染临时邮箱卡片。

**根因分析：**
1. `renderGroupList()` 未过滤临时邮箱分组
2. 虽已在 `selectGroup()` 中添加了跳转逻辑（`navigate('temp-emails')`），但分组条目仍可见，且可能被缓存的旧JS影响

**修复方案：**
- 在 `groups.js` 的 `renderGroupList()` 中过滤掉 `name === '临时邮箱'` 的分组
- 确保临时邮箱的管理完全在独立页面 `#page-temp-emails` 中进行

---

### BUG-014：审计日志页面永远显示"加载中" 🔴 HIGH

**现象：** 点击侧边栏"审计日志"菜单，页面内容始终为 loading spinner。

**根因分析：**
1. `navigate()` 函数中没有审计日志页面的数据加载调用
2. 代码中完全没有 `loadAuditLogPage()` 函数
3. 后端 API `/api/audit-logs` 已存在（`outlook_web/routes/audit.py`）

**修复方案：**
- 在 `main.js` 中新增 `loadAuditLogPage()` 函数，调用 `/api/audit-logs` 并渲染到 `#auditLogContainer`
- 在 `navigate()` 中添加：`if (page === 'audit') loadAuditLogPage();`

---

### BUG-015：刷新日志页面永远显示"加载中" 🔴 HIGH

**现象：** 点击侧边栏"刷新日志"菜单，页面内容始终为 loading spinner。

**根因分析：**
1. `navigate()` 函数中没有刷新日志页面的数据加载调用
2. 已有 `loadRefreshLogs()` 函数（第 1806 行），但它渲染到 `#refreshLogsContainer` / `#refreshLogsList`（这是弹窗内元素，不是页面容器）
3. 页面容器 ID 是 `#refreshLogContainer`（少了个 s）

**修复方案：**
- 新增 `loadRefreshLogPage()` 函数，渲染到 `#refreshLogContainer`
- 在 `navigate()` 中添加：`if (page === 'refresh-log') loadRefreshLogPage();`
