# Bug 列表

## BUG-001: Toast 通知位置不正确 ✅ 已修复

**优先级**: P1 - 高

**描述**:
所有 Toast 通知弹窗显示在屏幕正中间，遮挡用户视线，影响操作体验。

**期望行为**:
所有通知统一显示在屏幕右下角（类似常见桌面通知位置）。

**修复方案**:
修改 `static/css/main.css` 中的 `.toast` 样式，定位到右下角。

---

## BUG-002: 切换分组时全选状态被取消 ✅ 已修复

**优先级**: P1 - 高

**描述**:
用户在默认分组中全选了邮箱，切换到另一个分组后，之前选中的邮箱状态丢失。

**修复方案**:
使用全局 `selectedAccountIds` Set 存储选中状态，切换分组时保持。

---

## BUG-003: 批量操作缺少删除邮箱功能 ✅ 已修复

**优先级**: P1 - 高

**描述**:
批量操作栏缺少"删除邮箱"功能。

**修复方案**:
1. 在 `routes/accounts.py` 中注册 `/api/accounts/batch-delete` 路由
2. 批量操作栏添加"删除"按钮
3. 实现 `batchDeleteAccounts()` 函数

---

## BUG-004: 验证码/链接提取功能异常 ✅ 已修复

**优先级**: P0 - 紧急

**描述**:
验证码提取 API 只使用 Graph API，未复用 Graph→IMAP 回退策略。

**修复方案**:
在 `api_extract_verification` 中添加 IMAP 回退支持（IMAP_SERVER_NEW 和 IMAP_SERVER_OLD）。

---

## BUG-005: 自动轮询开关类型不一致 ✅ 已修复

**优先级**: P0 - 紧急

**描述**:
后端返回 `enable_auto_polling` 是 boolean，前端用字符串比较 `=== 'true'`，导致判断永远为 false。

**修复方案**:
前端统一使用兼容判断：`data.settings.enable_auto_polling === true || data.settings.enable_auto_polling === 'true'`

---

## BUG-006: 自动轮询未满足 PRD 验收 ✅ 已修复

**优先级**: P1 - 高

**问题描述**:
1. 通知内容不符合"邮箱地址 + 邮件主题"
2. 红点位置不正确（应该在账号列表项上）
3. 轮询失败不停止且不提示
4. 静默轮询不满足（启动/停止都 showToast）

**修复方案**:
1. `showNewEmailNotification()` 增加邮箱地址和邮件主题
2. 添加 `showAccountNewEmailDot()` 在账号列表项上显示红点
3. 添加 `pollingErrorCount` 连续错误计数，超过阈值自动停止并提示
4. `startPolling()` 和 `stopPolling()` 改为静默模式

---

## BUG-007: 轮询次数校验与提示不一致 ✅ 已修复

**优先级**: P2 - 中

**描述**:
UI 提示"0 表示持续轮询"，但前后端校验都限制 1-100。

**修复方案**:
前后端校验改为 0-100，0 表示持续轮询。

---

## BUG-008: 复制按钮缺少加载/防重复点击 ✅ 已修复

**优先级**: P2 - 中

复制按钮缺少加载状态、防重复点击、成功/失败视觉反馈。
