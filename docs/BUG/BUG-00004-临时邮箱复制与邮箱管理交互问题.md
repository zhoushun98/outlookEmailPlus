# BUG-00004 — 临时邮箱复制与邮箱管理页交互问题

## 问题清单

### BUG-020: 临时邮箱页面顶栏邮箱名称不可点击复制 (MEDIUM)

**现象描述**:
在临时邮箱独立页面(`#page-temp-emails`)，当选中一个临时邮箱后，右侧消息列顶栏显示邮箱地址（`#tempEmailCurrentName`），但这个邮箱名称**不可点击复制**。

对比参考：在邮箱管理页面的三列布局中，右侧邮件列有一个 `#currentAccountBar`，里面的邮箱地址（`#currentAccountEmail`）可以通过 `onclick="copyCurrentEmail()"` 点击复制。但临时邮箱独立页面的 `#tempEmailCurrentName` 只是普通 `<span>` 文字，没有点击复制功能。

**期望行为**: 点击临时邮箱页面顶栏的邮箱名称，应自动复制该邮箱地址到剪贴板并弹出提示。

**涉及代码**:
- `templates/index.html` 第 278 行: `<span class="column-title" id="tempEmailCurrentName">` — 缺少 onclick 和 cursor:pointer
- `static/js/features/temp_emails.js` `selectTempEmail()` 第 148 行: `tempName.textContent = email` — 只设置文字，没有绑定点击事件

### BUG-021: 邮箱管理页进入时不会自动选中默认分组 (MEDIUM)

**现象描述**:
点击侧边栏进入"邮箱管理"页面后，左侧分组列表加载出来了，但**没有自动选中任何分组**。账号列仍显示"请从左侧选择一个分组"的空状态提示。

尽管"临时邮箱"分组已被过滤（不显示在列表中），但默认分组并未被自动选中。用户必须手动点击"默认分组"才能看到账号列表。

**期望行为**: 进入邮箱管理页面时，如果没有已选中的分组（`currentGroupId` 为空），应自动选中第一个非临时邮箱分组（通常是"默认分组" id=1），并自动加载该分组的账号。

**涉及代码**:
- `static/js/features/groups.js` `loadGroups()` 第 24-35 行: 只在 `currentGroupId` 已有值时保持选中，否则什么都不做
- 需要增加: 当 `currentGroupId` 为空时，自动选中第一个非临时邮箱分组
- **状态**: ✅ 已修复 — `loadGroups()` 增加 else 分支自动选中第一个非临时邮箱分组

---

### BUG-022: 切换分组时 currentAccountBar（当前邮箱栏）不会重置/隐藏 (MEDIUM)

**现象描述**:
在邮箱管理页面，选中某分组并点击一个账号后，右侧邮件列顶部显示"当前邮箱：xxx@xxx.com"的 `currentAccountBar`。但当**切换到另一个分组**时，虽然账号列表刷新了，但 `currentAccountBar` 仍然显示之前的邮箱地址，不会隐藏或更新。

这导致用户切换分组后，看到的"当前邮箱"信息与实际选中状态不一致。

**期望行为**: 切换分组时，`currentAccountBar` 应该被隐藏（`display:none`），邮件列表回到空状态"请从左侧选择一个邮箱账号"，直到用户在新分组中重新选择一个账号。

**涉及代码**:
- `static/js/features/groups.js` `selectGroup()` 第 81-122 行: 切换分组时没有重置 currentAccountBar 和邮件列
- 需要增加: 在 `selectGroup()` 中隐藏 `currentAccountBar`，清空 `emailList`，隐藏 `emailDetailSection`
- **状态**: ✅ 已修复 — `selectGroup()` 开头增加完整的邮件列 UI 重置（6 个元素）

## 修复优先级

1. **BUG-022** (切换分组不重置) — 影响日常使用体验，信息不一致容易误导用户
2. **BUG-021** (不自动选中默认分组) — 影响首次进入体验
3. **BUG-020** (临时邮箱顶栏不可复制) — 功能缺失但有替代方案（点卡片上的邮箱地址）
