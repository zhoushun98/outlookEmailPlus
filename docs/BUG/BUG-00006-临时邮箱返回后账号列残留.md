## 问题清单

### BUG-024: 从临时邮箱页返回邮箱管理后，账号列显示临时邮箱数据残留 (HIGH)

**现象描述**:
1. 用户首次进入邮箱管理页面，默认分组自动选中，账号列正常显示该分组的邮箱账号
2. 用户点击侧边栏"临时邮箱"导航到临时邮箱页面
3. 用户再点击侧边栏"邮箱管理"回到邮箱管理页面
4. **中间的账号列 `#accountList` 仍然显示临时邮箱的卡片数据**，而非当前分组的正式邮箱账号

**根因分析**:

`#accountList` 容器被临时邮箱和普通邮箱**共用**：

1. `renderTempEmailList()` 在 `temp_emails.js` 中同时写入两个容器：
   - `document.getElementById('accountList')` → 邮箱管理页的账号列
   - `document.getElementById('tempEmailContainer')` → 临时邮箱独立页

2. `navigate('mailbox')` 在第二次进入时检查 `groups.length === 0`，由于分组已加载过（length > 0），**跳过 `loadGroups()`**

3. `loadGroups()` 中自动选中默认分组的逻辑仅在 `currentGroupId` 为空时执行，但切换过页面后 `currentGroupId` 已有值

4. 结果：没有任何代码去刷新 `#accountList` 的内容，临时邮箱数据残留

**修复方案**:
在 `navigate()` 中，当返回 mailbox 页面且 groups 已加载时，重新加载当前分组的账号：
```javascript
if (page === 'mailbox') {
    if (groups.length === 0) {
        loadGroups();
    } else if (currentGroupId) {
        loadAccountsByGroup(currentGroupId);
    }
}
```

**涉及代码**:
- `static/js/main.js` `navigate()` 函数第 69 行

**状态**: ✅ 已修复 — `navigate('mailbox')` 改为：首次加载 loadGroups()，返回时 loadAccountsByGroup(currentGroupId)
