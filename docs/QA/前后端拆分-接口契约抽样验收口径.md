# 前后端拆分｜接口契约抽样验收口径（最小字段契约）

- 日期：2026-02-23
- 目的：为“前后端拆分与模块化”提供可执行的 **抽样验收口径**，确保重构只改变代码组织，不改变对外行为。
- 说明：这里定义的是“最小字段契约”（存在性与基础结构），不穷举全部字段。

---

## 1) 抽样接口清单（建议每阶段必测）

### 1.1 基础可用性

- `GET /healthz`
  - 期望：`200`
  - JSON：`{ "status": "ok" }`

### 1.2 登录与鉴权

- `POST /login`（JSON：`{"password":"..."}`）
  - 期望：`200`
  - JSON：`{ "success": true }`（成功时）
- 任一需要登录的接口（例如 `GET /api/system/health`）
  - 未登录期望：`401`
  - JSON：`{ success:false, need_login:true, error:{code,status,trace_id,...} }`

### 1.3 系统与运维抽样

- `GET /api/system/health`（登录后）
  - 期望：`200`
  - JSON：
    - `success=true`
    - `health.service / health.database`
    - `health.scheduler.enabled / health.scheduler.autostart`
    - `health.refresh.locked / health.refresh.running`

- `GET /api/scheduler/status`（登录后）
  - 期望：`200`
  - JSON：
    - `success=true`
    - `scheduler.*`（至少包含 `enabled/autostart/heartbeat_age_seconds`）
    - `refresh.*`（至少包含 `last_scheduled/running/lock`）

- `GET /api/system/upgrade-status`（登录后）
  - 期望：`200`
  - JSON：
    - `success=true`
    - `upgrade.schema_version/upgrade.target_version/upgrade.up_to_date`
    - `upgrade.backup_hint`

### 1.4 基础业务抽样（不触发外网）

- `GET /api/groups`（登录后）
  - 期望：`200`
  - JSON：`{success:true, groups:[...] }`

- `GET /api/tags`（登录后）
  - 期望：`200`
  - JSON：`{success:true, tags:[...] }`

- `GET /api/accounts`（登录后）
  - 期望：`200`
  - JSON：`{success:true, accounts:[...] }`

- `GET /api/audit-logs`（登录后）
  - 期望：`200`
  - JSON：`{success:true, logs:[...], total, limit, offset }`
