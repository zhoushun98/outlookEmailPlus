# 00002｜前后端拆分与模块化：回滚点与切换方式

- 日期：2026-02-23
- 目的：为 `docs/TODO/Outlook邮件管理工具-前后端拆分与模块化TODO.md` 的每个阶段定义**可回滚点**与**切换方式**，确保“渐进式拆分”发生问题时可以快速回退到上一稳定基线。
- 约束：保持 **URL/响应契约/错误结构/trace_id/调度器行为** 不回归。

---

## 总体策略（适用于所有阶段）

1. **每阶段必须可验证**：至少通过 `python -m unittest discover -s tests -v` 与抽样契约门禁（`tests/test_smoke_contract.py`）。
2. **每阶段必须可回滚**：发生问题时优先回退到“上一阶段的绿色基线”（Git 回退/恢复上一版本镜像/回滚发布包）。
3. **入口保持稳定**：对外部署入口始终保持 `web_outlook_app:app` 不变；内部允许引入兼容层，但要有明确删除时机（见 P1-7）。

---

## 阶段回滚点（按 TODO 阶段）

### 阶段 0：基线冻结（门禁）

- 产出：
  - 抽样契约口径：`docs/QA/前后端拆分-接口契约抽样验收口径.md`
  - 抽样契约/Smoke 测试：`tests/test_smoke_contract.py`
- 回滚点：
  - 以“门禁通过”为基线；后续任何重构若导致门禁失败，应直接回退到最近一次门禁通过的版本。
- 切换方式：
  - 无需切换；本阶段仅新增测试与文档。

### 阶段 1：后端骨架 + 入口兼容（create_app）

- 目标：
  - 引入 `create_app()`，并保持 `web_outlook_app:app` 作为稳定入口。
  - 收敛 import-time 副作用：初始化与调度器启动受控（可通过环境变量/运行模式关闭）。
- 回滚点：
  - 保留兼容层（`web_outlook_app.py` 作为 facade），出现问题时可回退到“单体入口版本”（上一绿色基线）。
- 切换方式（运行侧）：
  - 仍使用 `web_outlook_app:app` 启动，无需变更部署命令。
  - 调度器自启动由 `SCHEDULER_AUTOSTART=true|false` 控制（测试/诊断场景可关闭）。

### 阶段 2：基础设施模块化（config/errors/security/db/audit）

- 目标：
  - 拆出基础设施模块，但对外 API 行为保持一致。
- 回滚点：
  - 每次拆分以“模块抽取 + 兼容 re-export”为单位提交；若出现回归，优先回退到上一绿色基线。
- 切换方式（代码侧）：
  - 允许短期保留旧函数入口（`web_outlook_app.*` re-export），待 P1-7 再清理。

### 阶段 3：repositories/services 模块化

- 目标：
  - SQL 访问集中（repo），业务编排集中（service），并补齐单测。
- 回滚点：
  - 每条业务链路迁移完成必须附带测试用例；测试失败则回退到上一绿色基线。
- 切换方式：
  - 同阶段 2：兼容层保留，逐步替换内部调用指向新模块。

### 阶段 4：Routes 模块化（Blueprint，URL 不变）

- 目标：
  - 路由迁移到 Blueprint，保持 URL/方法/响应结构不变。
- 回滚点：
  - 建议“按领域路由模块”逐个迁移；任一 Blueprint 迁移失败可回退到上一绿色基线。
- 切换方式：
  - 迁移期允许保留旧路由实现（不对外暴露新 URL），通过“注册/不注册”切换新旧实现（仅内部开发阶段使用）。

### 阶段 5：前端静态资源拆分（零构建）

- 目标：
  - 从模板内联 CSS/JS 迁出到 `static/css` 与 `static/js`（零构建）。
  - 模板改为通过 `<link rel="stylesheet" ...>` 与 `<script src="..."></script>` 引用静态资源（不强制使用 ESModule，避免大规模重写与行为回归）。
- 回滚点：
  - 若外链资源加载/缓存导致页面不可用，优先回退到上一绿色基线（Git 回退/恢复上一版本镜像/回滚发布包）。
- 切换方式：
  - 当前未引入运行开关；如需要“运行侧快速切换内联/外链”，可在后续阶段增加开关（例如 `FRONTEND_EXTERNAL_ASSETS=true|false`），但需同步补齐门禁与回归验证。

### 阶段 6：文档与维护指南

- 目标：
  - README/开发者文档补齐模块职责、依赖方向、测试运行方式与迁移说明。
- 回滚点：
  - 文档变更不影响运行；但若文档与实现不一致，应以实现为准并尽快纠正文档。
