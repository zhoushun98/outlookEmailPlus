## 1. 文档目的

本 TDD 用于描述 UI 全局美化重设计所需的技术实现细节。

重点回答：

- 如何**完全不改动后端代码**，仅修改前端文件实现 UI 改造
- 如何保证**所有 JS 函数接口兼容**（DOM ID、函数名、全局变量名均不变）
- 如何从旧的**四栏拖拽布局**迁移到**固定三栏 + 侧边栏**布局
- 如何设计 CSS 变量体系支持**双主题（亮色/暗色）切换**
- 如何实现**现代国风**视觉设计系统
- **关键算法**（token 状态计算、账号头像颜色生成、验证码提取）的精确实现
- 哪些文件需要**完全重写**，哪些需要**部分修改**，哪些需要**删除**

## 2. 设计原则与硬约束

### 2.1 后端零改动原则

- 不修改任何 Python 文件（`outlook_web/`、`web_outlook_app.py`、`start.py`）
- 不修改任何 Flask 路由、controller、service、repository
- 不修改 `requirements.txt`、`Dockerfile`、`.env` 配置
- API 接口路径、请求格式、响应结构**一字不改**

### 2.2 JS 函数接口兼容性原则

`templates/index.html` 内联脚本和 `static/js/features/*.js` 中存在大量全局函数和 DOM 操作，重建 HTML 骨架时必须保证：

**所有被 JS 引用的 DOM element ID 不变（或在 JS 中同步更新）：**

| DOM ID | 当前位置 | 被哪些函数引用 |
|--------|---------|--------------|
| `currentAccount` | 顶栏 | `selectAccount()`, `copyCurrentEmail()` |
| `currentAccountEmail` | 顶栏 | `selectAccount()`, `copyCurrentEmail()` |
| `groupList` | 分组栏 | `renderGroupList()`, `loadGroups()` |
| `accountList` | 账号栏 | `renderAccountList()`, `loadAccountsByGroup()` |
| `emailList` | 邮件列表区 | `loadEmails()`, `renderEmailList()` |
| `emailDetail` | 邮件详情区 | `showEmailDetail()`, `selectAccount()` |
| `emailDetailToolbar` | 邮件详情工具栏 | `showEmailDetail()`, `selectAccount()` |
| `emailCount` | 邮件计数 | `loadEmails()`, `selectAccount()` |
| `methodTag` | API 方式标签 | `loadEmails()`, `selectAccount()` |
| `folderTabs` | 文件夹 Tab | `selectAccount()`, `loadEmails()` |
| `csrfToken` | meta tag | `initCSRFToken()` |
| `accountPanelTitle` | 账号栏标题 | `loadAccountsByGroup()` |

**所有 CSS 类名选择器不变（或在 JS/HTML 中同步更新）：**

| CSS 选择器 | 被哪些 JS 引用 |
|-----------|--------------|
| `.account-item` | `selectAccount()`, `renderAccountList()` |
| `.account-item.active` | `selectAccount()` |
| `.account-email` | `selectAccount()` |
| `.folder-tab` | `selectAccount()`, `loadEmails()` |
| `.folder-tab.active` | `loadEmails()` |
| `.refresh-btn` | `loadEmails()` |
| `.loading`, `.loading-spinner` | `loadEmails()`, `loadGroups()` |

### 2.3 渐进可回滚原则

- 每个 Phase 完成后必须可以独立运行并通过烟雾测试
- 旧版 CSS 和 HTML 在实施前保留 Git 快照

### 2.4 技术栈不变原则

- 不引入任何 CSS 框架（Bootstrap、Tailwind）
- 不引入任何 JS 框架（React、Vue、jQuery）
- 所有实现用原生 HTML + CSS + JavaScript

│   │  ├── state-manager.js: 主题持久化（简化保留）          │  │
│   │  └── [删除] layout-manager.js, layout-bootstrap.js   │  │
│   └──────────────────────────────────────────────────────┘  │
│                            ↕ fetch API                       │
│   Backend (Flask - 不改动)                                   │

### 3.2 新布局结构 vs 旧布局结构

**旧布局（四栏拖拽）：**
```
<nav class="navbar">       ← 顶栏横贯全宽
<div class="main-container">
  <div class="group-panel resizable-panel">   ← 可拖拽
  <div class="account-panel resizable-panel"> ← 可拖拽
  <div class="email-panel resizable-panel">   ← 可拖拽
  <div class="detail-panel">
```

**新布局（侧栏 + 三栏固定）：**
```
<aside id="app-sidebar">       ← 左侧固定导航栏（宽60px/220px）
<div class="main-wrapper">
  <header class="topbar">      ← 顶栏（仅占内容区）
  <div class="workspace">
    <div class="col-groups">   ← 固定宽度 220px
    <div class="col-accounts"> ← 固定宽度 280px
    <div class="col-email">    ← flex: 1（含邮件列表+详情）
      <div class="col-email-list">
      <div class="col-email-detail">
```

## 7. 核心算法详细设计

### 7.1 账号头像颜色生成算法（`stringToColor`）

#### 设计目标

- 同一邮箱地址每次生成颜色相同（稳定性）
- 相似邮箱颜色有区分度（分散性）
- 颜色饱和度和亮度在可读范围内（视觉质量）

#### 算法伪代码

```
function stringToColor(str):
    // 1. 计算字符串哈希
    hash = 0
    for each char in str:
        code = char.charCodeAt(0)
        hash = (hash << 5) - hash + code
        hash = hash & hash   // 保持 32 位整数

    // 2. 提取色相（0-360）
    hue = Math.abs(hash) % 360

    // 3. 构建渐变（固定饱和度70%，亮度50%，模拟现代国风）
    hue2 = (hue + 30) % 360  // 第二色相偏移30度
    gradient = `linear-gradient(135deg, hsl(${hue},65%,55%), hsl(${hue2},70%,45%))`

    // 4. 计算文字颜色（深色背景用白色文字）
    lightness = 50  // 固定亮度中间值
    textColor = lightness > 55 ? '#2C1810' : '#FFFFFF'

    return { gradient, textColor }
```

#### JavaScript 实现

```javascript
function stringToColor(str) {
    let hash = 0;
    for (let i = 0; i < str.length; i++) {
        hash = (hash << 5) - hash + str.charCodeAt(i);
        hash |= 0;
    }
    const hue = Math.abs(hash) % 360;
    const hue2 = (hue + 30) % 360;
    return {
        gradient: `linear-gradient(135deg, hsl(${hue},65%,55%), hsl(${hue2},70%,45%))`,
        textColor: '#FFFFFF'
    };
}
```

#### 边界情况

| 输入 | 处理方式 |
|------|---------|
| `null` / `undefined` | 用默认色 `--clr-primary` |
| 空字符串 `""` | `hash=0` → 固定色相 0（红色系） |
| 单字符 `"a"` | 正常计算 |
| 含 Unicode 字符 | `charCodeAt()` 可处理，无需特殊处理 |

### 7.2 Token 状态计算算法（`getTokenStatus`）

#### 接口契约（基于 `/api/accounts` 响应字段）

需要使用的账号字段：

| 字段名 | 类型 | 含义 |
|--------|------|------|
| `token_valid` | `boolean \| undefined` | Token 是否有效 |
| `token_expires_at` | `string \| undefined` | ISO 8601 过期时间，如 `"2026-04-01T10:00:00"` |
| `token_status` | `string \| undefined` | 可选状态字符串（部分情况下有） |

#### 算法伪代码

```
function getTokenStatus(account):
    // 1. 字段缺失时返回未知
    if account.token_valid === undefined:
        return { status: 'unknown', label: '? 未知', cssClass: 'unknown' }

    // 2. 明确无效
    if account.token_valid === false:
        return { status: 'expired', label: '✗ 已过期', cssClass: 'expired' }

    // 3. 有效 — 检查是否即将过期（30天内）
    if account.token_expires_at:
        expiresDate = new Date(account.token_expires_at)
        now = new Date()
        daysLeft = (expiresDate - now) / (1000 * 60 * 60 * 24)

        if daysLeft < 0:
            return { status: 'expired', label: '✗ 已过期', cssClass: 'expired' }
        if daysLeft <= 30:
            return { status: 'expiring', label: '⚠ 即将过期', cssClass: 'expiring' }

    return { status: 'valid', label: '✓ 有效', cssClass: 'valid' }
```

#### 边界情况

| 情况 | 处理 |
|------|------|
| `token_expires_at` 为无效日期字符串 | `new Date()` 返回 `Invalid Date`，`NaN` 比较结果为 false，降级为 `valid` |
| `token_expires_at` 缺失但 `token_valid=true` | 直接返回 `valid`（不估算剩余天数） |
| `token_expires_at` 已过期但 `token_valid=true` | 以 `token_expires_at` 为准，返回 `expired`（数据一致性）|

### 7.3 验证码提取算法（`extractAndHighlightVerifCode`）

#### 设计目标

从邮件文本中识别最可能是验证码的数字序列，用 `<mark>` 高亮展示。

#### 匹配规则（优先级从高到低）

| 规则 | 正则 | 说明 |
|------|------|------|
| 6位纯数字（最常见） | `/\b(\d{6})\b/g` | OTP、SMS 验证码 |
| 4位纯数字 | `/\b(\d{4})\b/g` | 短验证码 |
| 8位纯数字 | `/\b(\d{8})\b/g` | 长验证码 |
| 字母数字混合 6-8位 | `/\b([A-Z0-9]{6,8})\b/g` | 激活码 |
| 短横线分隔 | `/\b(\d{3}-\d{3})\b/g` | 如 123-456 |

#### 算法伪代码

```
function extractAndHighlightVerifCode(text):
    if not text: return ''

    // 安全净化
    safeText = DOMPurify.sanitize(text)

    // 按优先级尝试匹配
    patterns = [
        { regex: /\b(\d{6})\b/g,       tag: 'verif-code' },
        { regex: /\b(\d{4})\b/g,        tag: 'verif-code' },
        { regex: /\b(\d{8})\b/g,        tag: 'verif-code' },
        { regex: /\b([A-Z0-9]{6,8})\b/g, tag: 'verif-code' },
        { regex: /\b(\d{3}-\d{3})\b/g,  tag: 'verif-code' }
    ]

    for pattern in patterns:
        if pattern.regex.test(safeText):
            return safeText.replace(pattern.regex, '<mark class="${pattern.tag}">$1</mark>')

    return safeText
```

#### 边界情况

| 情况 | 处理 |
|------|------|
| 文本中有多个数字序列 | 全部高亮（`/g` 全局匹配） |
| 文本为 HTML（富文本邮件） | 先经 `DOMPurify.sanitize()` 再做正则 |
| `DOMPurify` 未加载 | 降级为纯文本 `textContent`（避免 XSS） |

## 11. 性能优化策略

### 11.1 CSS 渲染性能

| 策略 | 实现方式 | 目标 |
|------|---------|------|
| 硬件加速过渡 | 仅对 `transform`, `opacity` 使用 transition | 60fps 动画 |
| 避免重排 | 使用 `transform: translateX()` 代替修改 `width` | 侧边栏展开动画 |
| CSS 变量主题切换 | 修改 `data-theme` 属性，CSS 变量自动级联 | 零 JS 重排 |
| 列表渲染 | 批量拼接 HTML 字符串，一次 `innerHTML =` | 减少 DOM 操作次数 |

### 11.2 滚动性能

```css
.col-groups,
.col-accounts,
.col-email-list,
.col-email-detail {
  overflow-y: auto;
  /* 启用 GPU 合成层，避免主线程卡顿 */
  will-change: transform;
  /* iOS 动量滚动 */
  -webkit-overflow-scrolling: touch;
}
```

### 11.3 主题切换无闪烁（FOUC 防止）

主题初始化脚本必须在 `<head>` 末尾、所有 `<body>` 内容渲染前同步执行：

```html
<head>
  <!-- CSS 引用 -->
  <link rel="stylesheet" href=".../main.css">

  <!-- 关键：同步初始化主题，防止亮色→暗色闪烁 -->
  <script>
    (function(){
      const t = localStorage.getItem('outlook_theme') || 'light';
      document.documentElement.setAttribute('data-theme', t);
    })();
  </script>
</head>
```

### 11.4 渲染批处理

账号卡片和邮件列表使用字符串拼接后一次设置 `innerHTML`：

```javascript
// ✅ 正确：批量渲染
function renderAccountList(accounts) {
    const container = document.getElementById('accountList');
    // 一次性构建所有 HTML
    container.innerHTML = accounts.map(account => {
        const color = stringToColor(account.email);

        return `<div class="account-item" onclick="selectAccount('${account.email}')">
            <div class="account-avatar" style="background:${color.gradient}">
                ${account.email[0].toUpperCase()}
            </div>
            ...
        </div>`;
    }).join('');
}

// ❌ 避免：逐个 appendChild
```

## 12. 错误处理与边界情况

### 12.1 数据缺失降级策略

| 缺失字段 | 降级处理 |
|---------|---------|
| `account.email` 为空 | 头像显示 `?`，跳过点击事件 |
| `account.token_valid` 缺失 | `getTokenStatus()` 返回 `unknown` 状态 |
| `account.token_expires_at` 无效 | 降级为 `valid`（不显示过期）|
| 邮件 `subject` 为空 | 显示 `（无主题）` |
| 邮件 `from` 为空 | 显示 `未知发件人` |
| 邮件正文为空 | 详情区显示 `（此邮件没有内容）` |
| `DOMPurify` 未加载 | 用 `textContent` 赋值（兜底） |

### 12.2 API 失败处理

API 失败时，渲染函数按以下方式降级显示：

```javascript
// loadGroups() 失败时
container.innerHTML = `
  <div class="empty-state">
    <div class="empty-state-icon">⚠️</div>
    <div class="empty-state-text">加载失败，请刷新</div>
    <button onclick="loadGroups()">重试</button>
  </div>
`;
```

### 12.3 主题 localStorage 失败

隐私浏览模式或 localStorage 被禁用时：

```javascript
function toggleTheme() {
    const current = document.documentElement.getAttribute('data-theme') || 'light';
    const next = current === 'light' ? 'dark' : 'light';
    document.documentElement.setAttribute('data-theme', next);
    try {
        localStorage.setItem('outlook_theme', next);
    } catch (_) {
        // 静默失败，主题仍然切换，仅不持久化
    }
}
```
