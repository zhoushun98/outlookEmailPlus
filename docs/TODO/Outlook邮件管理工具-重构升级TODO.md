## P0（必须完成）

### 1) 运行与定时刷新可靠性

- [x] 交付“Web 服务运行”能力：管理界面与核心接口可用
- [x] 交付“后台刷新任务运行”能力：可在无人值守场景执行定时刷新
- [x] 明确并交付“官方推荐运行方式”，并保证定时刷新在该方式下可用且可验证
- [x] 定时刷新能力齐全：启用/停用、两类调度配置方式、配置校验、未来触发时间预览
- [x] 定时刷新可验证：最近一次触发时间、触发来源标识、触发结果统计（总数/成功/失败）
- [x] 刷新冲突控制：避免不可解释的重复执行；刷新中再次触发需有明确提示或拒绝策略

### 2) 状态一致性（重启/扩容下仍可用）

- [x] 导出二次验证凭证与有效期管理在重启/扩容下保持一致
- [x] 登录失败计数、锁定状态与解锁时间在重启/扩容下保持一致
- [x] 定时刷新最近触发信息与（如有）执行中状态在重启/扩容下保持一致
- [x] 所有上述状态具备过期与清理策略，并可被管理员诊断查看

### 3) 错误与追踪一致性

- [x] 所有关键接口返回统一的错误结构（含错误编号、中文提示、追踪标识、可选细节）
- [x] 错误信息默认脱敏，不泄露令牌、密码等敏感内容
- [x] 多方式回退失败时，能聚合展示各方式失败原因（中文可理解）
- [x] 未捕获异常统一处理：返回稳定错误结构，并可通过追踪标识定位

### 4) 数据兼容与升级保障

- [x] 升级后历史数据可继续使用（账号、分组、标签、设置、刷新历史、审计记录等）
- [x] 升级过程可验证：能识别升级是否完成；失败时给出明确提示与恢复建议（含追踪标识）
- [x] 升级前提供备份提示与可操作路径

### 5) 核心业务功能保真（不缩水且不回归）

- [x] 登录与会话：登录/退出/修改密码/失败限制/剩余等待提示
- [x] 分组管理：列表、增删改、颜色/描述、网络访问策略配置、系统分组保护
- [x] 账号管理：列表、增删改查、状态启停、备注、批量导入（校验+错误定位+结果汇总）、批量移动分组
- [x] 标签管理：标签增删、账号批量打/去标签、按标签筛选与组合筛选
- [x] 邮件读取：三类文件夹、分页/滚动加载、详情、全屏、信任模式提示、默认安全展示
- [x] 邮件删除：单封/批量、二次确认、失败原因分类提示、支持重试
- [x] Token 刷新：单账号刷新、全量刷新（进度反馈）、仅刷新失败、失败重试、结果记录
- [x] 刷新日志与统计：近半年查询、失败列表入口、统计展示、过期记录清理
- [x] 导出：按分组/全量/选择导出、二次验证、文件命名规则、导出审计记录
- [x] 审计：对导出/删除/修改账号/修改分组/修改设置等敏感操作可追溯
- [x] OAuth2 辅助：授权链接生成、凭据换取、常见失败原因提示、相关配置校验提示
- [x] 临时邮箱：生成、列表、邮件列表/详情、删除/清空、失败原因分类提示
- [x] 设置中心：修改登录密码/临时邮箱凭据/刷新策略；变更生效时机明确；高风险项有提示
